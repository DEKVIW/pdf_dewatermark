# 打包与分发

> **基准版本**：0.2.3  
> **主交付**：绿色目录 onedir + zip  
> **备选**：Inno Setup 安装包（非必做）  
> **本文定位**：发版操作说明 + **打包技术学习笔记**（用了啥、为什么）

GUI 实现见 `07-GUI开发说明.md`；本文不展开界面代码。

## 1. 产品决策（先定「发什么」）

| 问题 | 结论 | 原因（学习向） |
| --- | --- | --- |
| 单文件 `onefile`？ | **不首选** | 启动时解压到临时目录，慢；杀软更爱拦；出问题难查缺哪个 DLL |
| 目录 `onedir`？ | **主路径** | exe + `_internal` 同目录，启动快、结构透明，适合绿色分发 |
| 只发 zip？ | **是，主推** | 无安装权依赖；与「桌面工具」场景匹配；构建链简单 |
| Inno 安装包？ | **可选备选** | 要开始菜单/卸载项时再编；需本机装 Inno；不是功能完成度门槛 |
| 版本策略 | **最新覆盖 + releases 归档** | `dist/JingYe` 日常用；`dist/releases/JingYe-x.y.z` 可回滚 |

| 给谁 | 给什么 |
| --- | --- |
| 教程读者 / 自用 / 快速分发 | `JingYe-x.y.z.zip` 或 `dist/JingYe/` |
| 强依赖「安装向导」的用户 | （可选）`JingYe-Setup-x.y.z.exe` |

两种形态**程序本体相同**（都是同一套 onedir），差别只在「怎么落到用户磁盘」。

## 2. 技术栈总览（打包相关）

把「开发时的 Python 项目」变成「用户双击就能跑的 Windows 目录」，本项目用到的技术如下。

```text
┌─────────────────────────────────────────────────────────────┐
│  源码 (src/pdf_dewatermark) + packaging/gui_entry.py         │
│  依赖：PySide6 / qfluentwidgets / PyMuPDF / NumPy / Pillow  │
└────────────────────────────┬────────────────────────────────┘
                             │ PyInstaller 6
                             │  （Analysis → PYZ → EXE → COLLECT）
                             ▼
┌─────────────────────────────────────────────────────────────┐
│  dist/JingYe/   onedir 绿色包                                │
│  · JingYe.exe     引导加载器（无控制台窗口）                 │
│  · _internal/     解释器 + 依赖 DLL/pyd + 资源               │
└────────────────────────────┬────────────────────────────────┘
                             │ write_dist_meta.py / make_release_zip.py
                             ▼
┌─────────────────────────────────────────────────────────────┐
│  dist/releases/JingYe-x.y.z/ + JingYe-x.y.z.zip              │
│  （可选）Inno Setup ISCC → JingYe-Setup-x.y.z.exe            │
└─────────────────────────────────────────────────────────────┘
```

| 技术 / 组件 | 角色 | 在本项目中的位置 |
| --- | --- | --- |
| **Python 3.12 + venv** | 运行与打包环境 | `.venv`，必须含 GUI 依赖 |
| **setuptools / pyproject** | 可编辑安装、`__version__`、extras | `pip install -e ".[gui,packaging]"` |
| **PyInstaller** | 依赖分析 + 打 Windows 目录包 | `packaging/pdf_dewatermark_gui.spec` |
| **PyInstaller Hooks** | 自动收 Qt/NumPy/PIL 等二进制 | 构建日志里的 `hook-PySide6` 等 |
| **Bootloader (`runw.exe`)** | 无控制台 GUI 入口进程 | `console=False` 选用 windowed |
| **PowerShell 编排** | 一键：装 PyInstaller → 打包 → 元数据 → 归档 | `scripts/build_gui_onedir.ps1` |
| **Python 写元数据** | UTF-8 BOM 中文文件名/正文 | `packaging/write_dist_meta.py` |
| **zipfile** | 带中文路径的 zip | `packaging/make_release_zip.py`（不用 `Compress-Archive`） |
| **Inno Setup 6（可选）** | 把 onedir 再包成 Setup.exe | `packaging/jingye_setup.iss` + `build_installer.ps1` |

### 2.1 应用本身 vs 打包工具（别混）

| 层次 | 技术 | 用途 |
| --- | --- | --- |
| **应用运行** | PySide6、Fluent Widgets、PyMuPDF、NumPy、Pillow | 界面与 PDF/图像处理 |
| **打包** | PyInstaller + 脚本 | 把应用+依赖收成可分发目录 |
| **安装外壳（可选）** | Inno Setup | 复制目录、快捷方式、卸载，不改业务逻辑 |

学打包时重点看 **PyInstaller 如何发现依赖、如何生成 onedir**；学业务时看 `src/`。

### 2.2 onedir 运行时大致发生什么

1. 用户双击 `JingYe.exe`（PyInstaller **bootloader**）。  
2. Bootloader 找到同目录 `_internal`，设置路径，加载嵌入的 Python 与依赖。  
3. 执行入口 `packaging/gui_entry.py` → `pdf_dewatermark.gui.app:main`。  
4. `paths.get_app_root()` 在打包后指向 **exe 所在目录**，于是 `data/`、`output/` 落在绿色包旁。

因此：**exe 与 `_internal` 必须成套**，不能只拷一个 exe。

### 2.3 为何不用 onefile 作主方案

| onefile | onedir（本项目） |
| --- | --- |
| 单文件好看 | 多文件，需整夹分发 |
| 每次启动解压 → 慢、占临时盘 | 直接从 `_internal` 加载 → 快 |
| 杀软误报率往往更高 | 相对好查、好排除 |
| 适合极简单 CLI | 适合带 Qt 的桌面 GUI |

### 2.4 包为什么这么大（正常现象）

onedir 体积大头通常是：

- **Qt / PySide6**（GUI + 插件）  
- **PyMuPDF**（`mupdf` 原生库）  
- **NumPy**（含 BLAS 等）  
- Pillow 与其它 pyd/DLL  

这是「自带运行时」的代价，不是 zip 没压好。业务 PDF 样本、`legacy/` 不应打进包。

## 4. PyInstaller 配方学习（`pdf_dewatermark_gui.spec`）

`.spec` 是 Python 脚本，描述「从哪进、带什么、排出什么、产物叫啥」。

### 4.1 入口

```python
# packaging/gui_entry.py
from pdf_dewatermark.gui.app import main
if __name__ == "__main__":
    main()
```

Analysis 的脚本列表指向该文件，而不是直接扫整个包再猜入口。`pathex` 含 `src/`，保证能 `import pdf_dewatermark`。

### 4.2 流水线四段（概念）

| 阶段 | 宏/步骤 | 作用 |
| --- | --- | --- |
| **Analysis** | 静态分析 import 图；收集二进制、数据文件；应用 hooks | 决定包进什么 |
| **PYZ** | 纯 Python 模块打进压缩归档 | 业务代码主体 |
| **EXE** | 生成 `JingYe.exe`；`exclude_binaries=True` 表示依赖不进单文件 | onedir 的「壳」 |
| **COLLECT** | 把 exe + binaries + datas 收成 `dist/JingYe/` | 最终绿色目录 |

### 4.3 本 spec 里值得读的字段

| 配置 | 本项目取值 | 学习要点 |
| --- | --- | --- |
| `datas` | `user_README.txt`、GUI `resources/` | 非 import 的资源要手动声明 |
| `hiddenimports` | `PySide6.*`、`qfluentwidgets`、`fitz`、`PIL`、`numpy`、若干 `pdf_dewatermark.*` | 动态导入/插件式加载时分析器容易漏，显式列出 |
| `excludes` | `tkinter`、`matplotlib`、`notebook` | 减小体积、避免无关依赖 |
| `console=False` | 无黑色控制台窗口 | GUI 用 `runw` bootloader |
| `icon=` | `packaging/app.ico` | Windows 资源管理器图标 |
| `upx=True` | 尝试 UPX 压缩部分二进制 | 有环境则压；失败可关；与杀软策略有关 |
| `name="JingYe"` | 目录与 exe 名 | 产品英文名，路径用 ASCII 更稳 |

### 4.4 Hooks 是什么

PyInstaller 对常见库有 **hook 脚本**（如 `hook-PySide6.py`）：告诉打包器还要拷哪些 Qt 插件 DLL、平台插件路径等。  
构建日志里大量 `Processing standard module hook ...` 即此。学打包时：

- 缺 Qt 平台插件 → 查 PySide6 hook / 是否漏 plugins  
- 缺 `fitz` 原生库 → 查 PyMuPDF 是否进 binaries  

### 4.5 开发运行 vs 打包运行

| | 开发 | 打包后 |
| --- | --- | --- |
| 启动 | `python -m pdf_dewatermark` / `run_gui.bat` | `JingYe.exe` |
| 模块从哪来 | site-packages + src | `_internal` |
| 应用根目录 | 项目根 | exe 所在目录 |
| 依赖变更 | `pip install` 即可 | **必须重新 PyInstaller** |

## 5. 绿色包构建流水线（必做）

### 5.1 环境

```powershell
cd 项目根目录
.\.venv\Scripts\Activate.ps1
pip install -e ".[gui,packaging]"
```

- **`[gui]`**：PySide6、Fluent 等  
- **`[packaging]`**：`pyinstaller>=6`  
- 务必用 **装过 GUI 的同一 venv** 打包，否则 Analysis 会漏库或链到错误 Python  

### 5.2 一键命令

```powershell
powershell -File scripts/build_gui_onedir.ps1
```

### 5.3 `build_gui_onedir.ps1` 逐步在做什么

| 步骤 | 技术 | 说明 |
| --- | --- | --- |
| 1 | 定位项目根、优先 `.venv\Scripts\python.exe` | 避免系统 Python |
| 2 | `python -c "from pdf_dewatermark import __version__"` | 版本号唯一来源（与界面 branding 应同步） |
| 3 | `pip install pyinstaller` + `PyInstaller --noconfirm --clean spec` | 清缓存重分析，输出 `dist/JingYe` |
| 4 | `write_dist_meta.py dist/JingYe <ver>` | 写 VERSION/中文说明、建 data/output/logs |
| 5 | Python `shutil.copytree` → `dist/releases/JingYe-<ver>/` | 版本快照（路径尽量 ASCII） |
| 6 | `make_release_zip.py` → `JingYe-<ver>.zip` | 外发压缩包 |

脚本**刻意不用** PowerShell 字符串写中文文件名（见 §7）。

### 5.4 产物结构

```text
dist/
  JingYe/                         # 始终最新
    JingYe.exe
    _internal/                    # 解释器、Qt、MuPDF、NumPy…
    VERSION.txt                   # utf-8-sig，「净页 JingYe x.y.z」
    使用说明.txt                  # utf-8-sig
    README.txt                    # 同内容，ASCII 文件名备份
    app.ico
    data/  output/  logs/
  releases/
    JingYe-0.2.3/
    JingYe-0.2.3.zip              # ★ 主推外发
    JingYe-Setup-0.2.3.exe        # 可选 Inno 产物
```

### 5.5 版本号同步清单

| 位置 | 字段 |
| --- | --- |
| `src/pdf_dewatermark/__init__.py` | `__version__` |
| `src/pdf_dewatermark/gui/branding.py` | `APP_VERSION` |
| `pyproject.toml` | `project.version` |
| `packaging/jingye_setup.iss` | `#define MyAppVersion`（`build_installer.ps1` 可自动改） |
| `packaging/user_README.txt` / 文档 | 用户可见说明（按需） |

构建脚本**只读** `__version__`；改完版本再跑打包即可。

## 6. 元数据与 zip 技术点

### 6.1 `write_dist_meta.py`

- 用 **Python Unicode** 创建 `使用说明.txt`（避免 PS 5.1 编码坑）  
- 正文 **`utf-8-sig`（带 BOM）**，记事本可正确识别中文  
- 从 `user_README.txt` 拷贝说明；写 `VERSION.txt`  
- 创建 `data/`、`output/`、`logs/`；复制 `app.ico`  
- 清理历史乱码残留 txt  

### 6.2 `make_release_zip.py`

- 使用标准库 **`zipfile.ZIP_DEFLATED`**  
- 遍历目录写入 zip，相对路径 `as_posix()`  
- **不用** `Compress-Archive`：其对非 ASCII 文件名在部分 Windows 上不友好  

### 6.3 用户侧（绿色包）

1. 解压 `JingYe-x.y.z.zip` 到任意目录  
2. **整夹保留**（含 `_internal`）  
3. 双击 `JingYe.exe`  
4. 输出默认在包内 `output\`  

| 现象 | 处理 |
| --- | --- |
| 双击无反应 | 查杀软隔离；确认 `_internal` 完整 |
| 缺 DLL / 闪退 | 重新解压完整 zip，勿混版本 |
| 中文说明乱码 | 应用记事本打开 BOM 文件；用当前脚本重建包 |

## 7. 中文与 PowerShell：为什么单独用 Python 写文件

Windows PowerShell **5.1** 常见问题：

- `.ps1` 若按错误编码保存，脚本里的中文字面量会坏  
- 用错误编码 `Set-Content` 会写出「文件名已是乱码」的文件  

对策（本项目已采用）：

1. **编排脚本（.ps1）**：只写 ASCII 路径与命令  
2. **中文文件名与正文**：全部交给 **Python**（`utf-8` / `utf-8-sig`）  
3. zip 也用 Python，不用 `Compress-Archive`  

学到的通用原则：**跨工具链时，谁擅长 Unicode 谁写文件；Shell 只负责调度。**

## 8. 可选：Inno Setup 安装包

> **定位**：在已有 onedir **之上**再包一层安装体验。  
> **不是**替代 PyInstaller；未装 Inno 时**只发 zip 完全够用**。

### 8.1 Inno 是什么（学习向）

[Inno Setup](https://jrsoftware.org/isinfo.php) 是 Windows 上常用的**安装包制作工具**：

- 用 **`.iss` 脚本**描述：拷哪些文件、默认装哪、快捷方式、卸载  
- 编译器 **`ISCC.exe`** 生成单个 `Setup.exe`  
- 安装时把文件解到目标目录（本项目即整包 onedir），写卸载信息  

与 NSIS、WiX 同类；Inno 脚本可读、适合绿色目录「再包一层」。

### 8.2 本项目为何做成可选

| 已有 zip | 再上 Inno |
| --- | --- |
| 构建简单、教程分发够用 | 开始菜单、卸载项、向导感 |
| 用户解压即用 | 需本机装 Inno；签名后体验更好 |
| 主推 | 有明确「要安装包」需求再编 |

### 8.3 构建顺序

```powershell
# 1）必须先有绿色包
powershell -File scripts/build_gui_onedir.ps1

# 2）可选：编安装包（需 Inno Setup 6）
powershell -File scripts/build_installer.ps1
```

`build_installer.ps1`：

1. 检查 `dist\JingYe\JingYe.exe`  
2. 把 `.iss` 中 `MyAppVersion` 同步为 `__version__`  
3. 查找 `ISCC.exe`（PATH 或默认安装路径）  
4. 编译；找不到 ISCC 时 **exit 2**，并提示绿色包仍可用  

产物：`dist/releases/JingYe-Setup-<ver>.exe`。

### 8.4 `.iss` 要点（`jingye_setup.iss`）

| 配置 | 含义 |
| --- | --- |
| `Source: dist\JingYe\*` + `recursesubdirs` | 整包 onedir 拷进 `{app}` |
| `DefaultDirName={autopf}\JingYe` | 默认 Program Files 下 |
| `OutputBaseFilename=JingYe-Setup-{version}` | Setup 文件名 |
| `[Icons]` | 开始菜单、可选桌面快捷方式 |
| `[Run]` | 安装结束可启动 |
| `[UninstallDelete]` | 尽量只删空目录，保留用户 output/data 内容 |

语言：优先简体 `ChineseSimplified.isl`；若本机无语言包可改 `Default.isl`（向导英文，**程序仍是中文**）。

### 8.5 用户侧（安装包）

1. 运行 Setup（未签名时可能 SmartScreen 提示）  
2. 选目录 → 可选桌面图标 → 完成  
3. 卸载：应用和功能 / 开始菜单卸载项  

### 8.6 代码签名（未做）

Authenticode 签名可降低 SmartScreen 拦截；需证书 + `signtool`。  
**未签名也可以分发**；在说明中写清来源即可。签名与是否使用 Inno **无关**。

## 10. 学习路径建议

若目标是「看懂并会改打包」：

1. 读本文 §2 总览 + §4 `.spec`  
2. 本地跑一遍 `build_gui_onedir.ps1`，对照 `dist/JingYe/_internal` 里有哪些库  
3. 故意改 `hiddenimports` / `excludes` 看体积与能否启动（在分支上试）  
4. 读 `write_dist_meta.py`、`make_release_zip.py` 理解编码与 zip  
5. 有兴趣再装 Inno，对照 `.iss` 与官方文档学安装脚本  

官方文档：

- [PyInstaller 手册](https://pyinstaller.org/en/stable/)  
- [Inno Setup 文档](https://jrsoftware.org/ishelp/)
